By default, inactive JSPWiki sessions are logged out after 30 minutes. This value can be adjusted by adding the following entry to the JSPWiki's web.xml configuration file (where the session-timeout value is in
minutes):


60

This sets inactive sessions to be logged out after one hour.

Alternatively, a session timeout value can be configured in your application server. For example, on Payara Server go to server-config > Web Container, and under the Session Properties tab set the Session
Timeout value:

Tip: You can change the session timeout message displayed by JSPWiki when the timeout occurs by creating a wiki page named SessionExpired. It's also a good idea to add an [{ALLOW edit Admin}] ACL entry to this page.
